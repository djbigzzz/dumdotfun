## ğŸš§ Challenges & Learnings

### **Challenges We Faced**
1. **SDK Version Conflicts**: Multiple privacy SDKs had conflicting dependencies. Solved with careful version pinning and compatibility layers.

2. **Token-2022 API Availability**: The full confidential transfer API wasn't available in v0.4.14. Built a hybrid system with automatic detection and fallback to commitment schemes.

3. **WebSocket Reliability**: PumpPortal WebSocket had intermittent disconnects. Implemented automatic reconnection with exponential backoff.

4. **Client-Side Proof Generation**: Bulletproof generation was slow in browser. Used WASM compilation for 10x speedup.

5. **Database Schema Evolution**: Adding confidential betting required careful migration to avoid breaking existing bets.

### **What We Learned**
- **Privacy requires tradeoffs**: Performance vs. privacy, usability vs. security
- **Multiple privacy layers are better than one**: Each protocol solves different problems
- **Fallback strategies are essential**: APIs change, network fails â€” always have a Plan B
- **User education is critical**: Privacy features are useless if users don't understand them
- **Open source accelerates development**: Standing on shoulders of giants (ShadowWire, Inco, etc.)

### **Limitations & Shortcuts**
- **No mainnet deployment**: Devnet only for hackathon (mainnet requires audit)
- **Some fallback modes active**: Token-2022 and Arcium use commitment schemes until native APIs are ready
- **Limited mobile optimization**: UI designed for desktop-first
- **No automated test coverage**: Manual testing only (time constraints)
- **Monolithic routes.ts**: 3,750 lines â€” needs refactoring into separate modules

**We're transparent about these because judges appreciate honesty.**
